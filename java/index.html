Java Interview Questions with Detailed Answers (6–7 Years Experience)

1. Core Java

Q: What are the key features introduced in Java 8, 9, 11, and 17?

Java 8 introduced major changes with functional programming: Lambda expressions let you treat code as data; Stream API lets you process collections in a functional style; Functional interfaces like Predicate, Function; Optional helps avoid null checks; new Date/Time API (java.time) solves many design issues in java.util.Date.

Java 9 brought the Module System (Jigsaw) allowing better modularization and encapsulation. JShell enabled interactive experimentation. Flow API introduced reactive stream processing.

Java 11 removed older deprecated APIs, introduced HttpClient (supports async, sync HTTP calls), new string methods like strip(), repeat(), isBlank(), and improved performance/stability.

Java 17 added Sealed classes for controlled inheritance, Pattern matching for switch, Strong encapsulation of JDK internals, and became a long-term support (LTS) release.

Q: Explain the differences between == and .equals().

== checks if two references point to the same object in memory. It does not compare content. Use this for primitives or to verify reference equality.

.equals() checks for logical equivalence. For Strings or custom objects, override equals() to compare values.

String x = new String("hello");
String y = new String("hello");
System.out.println(x == y);       // false
System.out.println(x.equals(y));  // true
Q: How does Java handle memory management and garbage collection?

Java has automatic memory management using the Garbage Collector (GC). It manages heap memory where objects are allocated. The JVM identifies unreachable objects and cleans them to free space.

Memory is divided into Young Generation (Eden + Survivor spaces), Old Generation (tenured), and Metaspace. Popular GCs: Serial, Parallel, CMS (deprecated), G1 (default in Java 9+), ZGC, Shenandoah.

GC tuning and profiling tools include VisualVM, JConsole, GC logs, and Java Flight Recorder.

Q: What is the difference between HashMap, Hashtable, and ConcurrentHashMap?

HashMap is not thread-safe and allows one null key and multiple null values. It provides constant-time complexity O(1) for put/get in most cases.

Hashtable is thread-safe but uses coarse-grained synchronization, which causes performance bottlenecks.

ConcurrentHashMap is thread-safe and uses finer-grained locking (segment-level or bucket-level), allowing concurrent reads and controlled writes without locking the entire map.

Q: How does synchronized work in Java? What are its drawbacks?

synchronized blocks or methods lock access to prevent multiple threads from executing simultaneously on the same object. It guarantees atomicity and visibility.

Drawbacks include potential deadlocks, performance overhead due to lock contention, and difficulty in debugging concurrency bugs. Use ReentrantLock or java.util.concurrent utilities for advanced control.

Q: What is the difference between final, finally, and finalize()?
  • final: Declares constants, prevents method overriding, and class inheritance.
  • finally: Ensures a block of code runs regardless of exception, typically for resource cleanup.
  • finalize(): Called by GC before object deletion. Deprecated due to unpredictable behavior and alternatives like try-with-resources.
Q: What are the principles of OOP and how are they implemented in Java?
  • Encapsulation: Private fields with public getters/setters to protect data.
  • Abstraction: Hiding implementation details using interfaces and abstract classes.
  • Inheritance: Code reuse via extends. Enables hierarchical relationships.
  • Polymorphism: Method overloading (compile-time) and overriding (runtime) to provide flexibility.
abstract class Animal {
  abstract void speak();
}

class Dog extends Animal {
  void speak() { System.out.println("Bark"); }
}
Q: What is the difference between ArrayList and LinkedList?

ArrayList uses a dynamic array. It offers fast access (O(1)) by index but slow insertions/deletions (O(n)) in the middle.

LinkedList is a doubly linked list. It supports fast insertions/removals (O(1)) at head/tail but slow access (O(n)).

Use ArrayList when frequent random access is required, LinkedList for frequent inserts/removes.

Q: Explain method overloading vs method overriding.

Overloading: Same method name with different parameters in the same class. It provides compile-time polymorphism.

Overriding: Subclass provides specific implementation of a method defined in a superclass. It provides runtime polymorphism.

class Parent {
  void greet() { System.out.println("Hello"); }
}
class Child extends Parent {
  void greet() { System.out.println("Hi"); }
}
Q: What is the volatile keyword? When should it be used?

volatile ensures that the value of a variable is always read from main memory, not thread-local cache. It guarantees visibility but not atomicity.

Use when multiple threads read/write a single variable, and atomicity is not a concern (e.g., flags).

private volatile boolean running = true;
Q: How does autoboxing and unboxing work in Java?

Autoboxing: Automatic conversion of a primitive to its corresponding wrapper type.

Unboxing: Conversion from wrapper type to primitive.

Integer obj = 10; // autoboxing
int val = obj;   // unboxing

This simplifies code when using generics and collections.

Q: What are enums and how are they used?

enum defines a fixed set of constants. It improves type safety and readability.

enum Direction {
  NORTH, SOUTH, EAST, WEST
}
Direction d = Direction.NORTH;

Enums can have constructors, fields, and methods, making them more powerful than constants.

Q: What are static blocks and when are they executed?

Static blocks are used for static initialization of a class. They are executed once when the class is loaded into memory.

static {
  System.out.println("Static block executed");
}

Useful for complex static variable initialization.

Q: How does Java handle pass-by-value vs pass-by-reference?

Java is strictly pass-by-value. For primitives, it passes the actual value. For objects, it passes the value of the reference.

This means changes to object fields are reflected outside, but reassigning the object inside a method does not affect the original reference.

void changeName(Person p) {
  p.setName("John"); // modifies original object
  p = new Person();   // doesn't affect the caller
}
