Spring Boot Interview Questions

Core Spring Boot (Basics & Internals)
1. What is Spring Boot? How is it different from Spring?
  • Spring Boot is a project built on top of the Spring Framework to simplify the setup, configuration, and deployment of Spring applications.
  • It comes with embedded servers like Tomcat, Jetty, or Undertow, removing the need to deploy WAR files.
  • It minimizes the need for external configuration by offering default setups via auto-configuration.
  • Compared to Spring, it promotes rapid application development with opinionated starters and simplified project structure.
  • It reduces boilerplate code by auto-configuring components based on available classpath libraries.
2. Explain Spring Boot's auto-configuration mechanism.
  • Auto-configuration attempts to automatically configure your Spring application based on the JAR dependencies you have added.
  • It uses @EnableAutoConfiguration to look at the classpath and create beans automatically for various configurations.
  • Auto-configurations are conditionally loaded using @Conditional annotations (like @ConditionalOnClass, @ConditionalOnMissingBean).
  • It is enabled implicitly when using @SpringBootApplication.
  • Developers can override any configuration by defining their own beans in the context.
@EnableAutoConfiguration public class MyAppConfig { // Configuration beans or classpath detected settings are auto-configured here }
3. What is @SpringBootApplication? What annotations does it combine?
  • @SpringBootApplication is a convenience annotation that combines:
  • @Configuration: Marks the class as a source of bean definitions.
  • @EnableAutoConfiguration: Tells Spring Boot to auto-configure the application context.
  • @ComponentScan: Enables component scanning on the package of the annotated class.
  • It is typically placed on the main class to bootstrap a Spring Boot application.
@SpringBootApplication public class MyApplication { public static void main(String[] args) { SpringApplication.run(MyApplication.class, args); } } }
4. How does Spring Boot reduce boilerplate configuration?
  • Uses starter dependencies (like spring-boot-starter-web) to bundle commonly used libraries.
  • Auto-configures beans based on the dependencies present in the classpath.
  • Eliminates the need for verbose XML configuration files.
  • Supports externalized configuration via property or YAML files (application.properties / application.yml).
  • Includes development tools like devtools and Actuator for monitoring with minimal setup.
5. How do you create a Spring Boot application from scratch?
  • Use Spring Initializr (https://start.spring.io) to generate the base project with dependencies.
  • Choose your build tool (Maven or Gradle), language, and dependencies (e.g., Web, JPA).
  • Download the zip and import it into your IDE (e.g., IntelliJ or Eclipse).
  • Create a main class with @SpringBootApplication and a main method.
  • Run the application using the generated build file or directly from the IDE.
@SpringBootApplication public class HelloWorldApplication { public static void main(String[] args) { SpringApplication.run(HelloWorldApplication.class, args); } }
6. What is Spring Initializr? How do you use it?
  • A web-based project generator provided by Spring at https://start.spring.io.
  • Allows you to specify the language (Java/Kotlin/Groovy), build tool (Maven/Gradle), and dependencies.
  • Generates a complete base project with all necessary configuration and dependency files.
  • Speeds up project bootstrapping for new developers and teams.
  • Integrates well with popular IDEs like IntelliJ IDEA and Spring Tool Suite (STS).
// Visit https://start.spring.io // Choose project type (Maven/Gradle), language (Java/Kotlin/Groovy), dependencies (e.g., Web, JPA) // Click "Generate" to download your project skeleton
Spring Boot Configuration & Customization
7. How do you define and use external configurations in Spring Boot?
  • External configurations are defined in application.properties or application.yml.
  • You can define key-value pairs for ports, database settings, etc.
  • Use @Value or @ConfigurationProperties to bind configuration to beans.
  • Spring Boot supports different configuration sources like environment variables, command-line arguments, etc.
# application.properties server.port=8081 app.name=DemoApp # In Java @Value("${app.name}") private String appName;
8. What is application.properties vs application.yml?
  • application.properties uses flat key-value pairs.
  • application.yml supports hierarchical and structured configuration.
  • Both are functionally similar and are loaded by Spring Boot automatically.
  • YAML is more readable for complex nested configuration.
9. How do you use profiles (@Profile) in Spring Boot?
  • Use @Profile("dev") to activate beans for a specific environment.
  • Profiles are activated via spring.profiles.active in properties or as environment variables.
  • You can create separate property files like application-dev.properties.
  • Useful for environment-specific beans and configurations.
# application.properties spring.profiles.active=dev # application-dev.properties logging.level.org.springframework=DEBUG // Java Bean @Profile("dev") @Bean public DataSource devDataSource() { return new HikariDataSource(); }
10. How do you create custom properties and bind them using @ConfigurationProperties?
  • Create a POJO class with appropriate fields and getters/setters.
  • Annotate it with @ConfigurationProperties(prefix = "custom").
  • Register it as a bean using @Component or @EnableConfigurationProperties.
  • Define the properties in application.properties or application.yml.
# application.yml custom: username: admin timeout: 5000 // POJO @Component @ConfigurationProperties(prefix = "custom") public class CustomConfig { private String username; private int timeout; // getters and setters }
11. What are the differences between @Value and @ConfigurationProperties?
  • @Value is used for injecting individual properties.
  • @ConfigurationProperties binds a group of related properties to a bean.
  • @Value does not support complex nested structures easily.
  • @ConfigurationProperties is preferred for mapping structured configs and supports validation.
// Using @Value @Value("${server.port}") private int port; // Using @ConfigurationProperties @ConfigurationProperties(prefix = "server") public class ServerConfig { private int port; // getter/setter }
12. How to override default configuration of Spring Boot auto-configured beans?
  • Define a bean of the same type in your configuration class.
  • Spring Boot will prioritize your custom bean over the default one.
  • Use @ConditionalOnMissingBean to create fallbacks in auto-configurations.
  • Properties can also be overridden using external configuration files or environment variables.
13. What is the order of precedence in Spring Boot property resolution?
  • Command-line arguments
  • Java system properties (System.getProperties)
  • OS environment variables
  • Application properties files (application.yml/properties)
  • Default properties defined in @SpringBootApplication or SpringApplication
14. What is a custom @Conditional annotation?
  • It allows conditional bean registration based on custom logic.
  • Implemented using a class that implements Condition interface.
  • Use @Conditional(MyCustomCondition.class) on bean definitions.
  • Useful for advanced scenarios where registration depends on runtime checks.
public class MyCustomCondition implements Condition { @Override public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) { return context.getEnvironment().containsProperty("custom.enabled"); } } @Bean @Conditional(MyCustomCondition.class) public MyService myService() { return new MyService(); }
Dependency Injection & Bean Lifecycle
15. What are the different types of dependency injection in Spring Boot?
  • Constructor Injection – Preferred method; promotes immutability and testability.
  • Setter Injection – Useful for optional dependencies.
  • Field Injection – Less preferred; limits testability and immutability.
16. When should you use @Autowired vs constructor injection?
  • @Autowired can be used with constructors, setters, or fields.
  • Constructor injection is the recommended way as it enables final fields and easier testing.
  • Field injection hides dependencies and complicates unit testing.
17. What is the bean lifecycle in Spring?
  • Bean Instantiation – Container creates the bean instance.
  • Dependency Injection – Dependencies are injected by the container.
  • Bean Post Processing – Custom logic before/after initialization (BeanPostProcessor).
  • Initialization – @PostConstruct or afterPropertiesSet() is called.
  • Destruction – @PreDestroy or destroy() is invoked during shutdown.
18. How do you define a custom BeanPostProcessor or BeanFactoryPostProcessor?
  • Implement BeanPostProcessor or BeanFactoryPostProcessor interface.
  • Override required methods like postProcessBeforeInitialization.
  • Register it as a Spring bean in your configuration class.
  • Used for modifying bean properties or behavior during initialization.
@Component public class CustomBeanPostProcessor implements BeanPostProcessor { @Override public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException { System.out.println("Before Initialization: " + beanName); return bean; } }
19. How do scopes like singleton, prototype, request, and session behave in Boot?
  • Singleton – Default scope; one instance per Spring container.
  • Prototype – A new instance is created every time it is requested.
  • Request – Scoped to an HTTP request (Web-aware contexts).
  • Session – Scoped to an HTTP session (Web-aware contexts).
  • Use @Scope("prototype") or @Scope("request") to specify scope.
Spring MVC & REST APIs
20. How do you create REST APIs in Spring Boot?
  • Use @RestController to define a controller where all methods return domain objects instead of views, making it suitable for REST APIs.
  • Define API endpoints using mapping annotations like @GetMapping, @PostMapping, @PutMapping, etc., each mapping to an HTTP method.
  • Return Java objects or collections directly from controller methods, which Spring Boot serializes into JSON using Jackson.
@RestController @RequestMapping("/api") public class MyController { @GetMapping("/hello") public String sayHello() { return "Hello World"; } }
21. What are @RestController, @Controller, and @ResponseBody?
  • @Controller marks a class as a Spring MVC controller, typically returning views (e.g., JSP or Thymeleaf).
  • @ResponseBody tells Spring to serialize the return value of a method directly into the HTTP response body instead of rendering a view.
  • @RestController is a meta-annotation that combines @Controller and @ResponseBody, used for creating REST APIs more conveniently.
22. How does @RequestParam differ from @PathVariable?
  • @RequestParam is used to extract query parameters from the URL, typically used for filtering, sorting, or passing optional inputs.
  • @PathVariable is used to extract values embedded in the URI path and is commonly used for uniquely identifying resources.
  • @RequestParam expects the key to be in the query string, while @PathVariable binds directly to placeholders defined in the URI.
@GetMapping("/users") public User getUser(@RequestParam String id) {...} @GetMapping("/users/{id}") public User getUserById(@PathVariable String id) {...}
23. How do you implement exception handling in Spring Boot REST APIs?
  • Use @ControllerAdvice to create a global exception handler that can catch exceptions thrown by any controller.
  • Define specific methods annotated with @ExceptionHandler to handle particular exception types and return appropriate HTTP responses.
  • Enhance client experience by sending meaningful error messages and status codes using ResponseEntity.
@ControllerAdvice public class GlobalExceptionHandler { @ExceptionHandler(ResourceNotFoundException.class) public ResponseEntity handleNotFound(ResourceNotFoundException ex) { return ResponseEntity.status(HttpStatus.NOT_FOUND).body(ex.getMessage()); } }
24. What is ResponseEntity and when do you use it?
  • ResponseEntity is a generic wrapper for an HTTP response, allowing you to specify the status code, headers, and response body explicitly.
  • It's useful when you need to customize the HTTP status or return headers alongside the body content.
  • Commonly used in exception handlers or when returning different statuses (e.g., 200 OK, 201 CREATED, 400 BAD REQUEST).
return ResponseEntity.status(HttpStatus.CREATED).body(newUser);
25. How do you use HATEOAS with Spring Boot?
  • Spring HATEOAS is a module that helps create RESTful APIs with hypermedia links to improve API discoverability.
  • Use EntityModel to wrap your response object and attach Link instances representing navigable endpoints.
  • Add Spring HATEOAS dependency to your project and use static methods like linkTo and methodOn to generate links.
EntityModel model = EntityModel.of(user); model.add(Link.of("/api/users/" + user.getId(), "self")); return model;
26. What are HttpMessageConverters?
  • HttpMessageConverters are used by Spring MVC to convert HTTP requests and responses into Java objects and vice versa.
  • Spring Boot auto-configures converters like JacksonHttpMessageConverter for JSON, and others for XML or plain text.
  • You can customize or register additional converters by extending WebMvcConfigurer and overriding configureMessageConverters().
Data Access (Spring Data JPA, JDBC)
27. What is Spring Data JPA? How does it simplify data access?
  • Spring Data JPA is a part of the Spring Data project that aims to simplify the implementation of JPA-based data access layers.
  • It provides a set of high-level abstractions for working with databases, reducing the amount of boilerplate code required.
  • Developers can define repository interfaces by extending Spring Data interfaces, and Spring automatically provides the implementation at runtime.
  • It supports query generation from method names, pagination, sorting, and auditing out of the box.
  • Integrates seamlessly with Hibernate and other JPA providers.
28. Difference between CrudRepository, JpaRepository, and PagingAndSortingRepository?
  • CrudRepository: Provides basic CRUD operations such as save, findById, findAll, deleteById. Ideal for simple use cases.
  • PagingAndSortingRepository: Adds support for pagination and sorting capabilities via methods like findAll(Pageable pageable).
  • JpaRepository: Extends both CrudRepository and PagingAndSortingRepository and adds more JPA-specific functionality like flushing the persistence context and batch deletes.
29. How do you handle custom queries in Spring Data?
  • Custom queries can be defined using method name conventions, such as findByEmailAndStatus, which Spring parses into appropriate JPQL queries.
  • Use the @Query annotation when method naming becomes too complex or insufficient for the query you want to express.
  • Supports named queries defined via @NamedQuery on entity classes, which are useful for static query definitions reused across the application.
  • Repositories can also be customized by writing implementation classes manually and linking them with the custom interface.
30. What is the use of @Query annotation?
  • The @Query annotation allows developers to write JPQL or native SQL queries directly within repository interfaces.
  • It bypasses the need for derived query methods, offering better control and flexibility for complex logic.
  • You can use named parameters (e.g., :status) or positional parameters (e.g., ?1).
  • Combined with nativeQuery=true, it allows writing raw SQL queries for performance-critical operations or database-specific logic.
@Query("SELECT u FROM User u WHERE u.status = :status") List findByStatus(@Param("status") String status);
31. How to handle transactions in Spring Boot?
  • Spring Boot supports declarative transaction management using the @Transactional annotation from Spring Framework.
  • You can annotate service layer methods or entire classes to ensure the operations execute within a transactional context.
  • If any runtime exception is thrown, the transaction is automatically rolled back; checked exceptions require manual configuration for rollback.
  • Transactions can be customized using attributes such as isolation level, propagation behavior, timeout, and rollback rules.
@Transactional public void updateAccountBalance(...) { // transactional logic }
32. How to use native SQL queries and projections?
  • Use @Query with nativeQuery = true to execute raw SQL queries for complex or performance-optimized queries.
  • Projections allow you to retrieve only a subset of entity attributes by defining DTO interfaces or classes.
  • Spring automatically maps the query result to the projection interface using getters matching the alias names in SQL results.
  • Reduces overhead by avoiding full entity loading, making it ideal for read-only views or API responses.
@Query(value = "SELECT name, email FROM users WHERE active = true", nativeQuery = true) List findActiveUserDetails(); interface UserProjection { String getName(); String getEmail(); }
Error Handling & Validation
33. How to handle exceptions globally using @ControllerAdvice?
  • Use @ControllerAdvice to define a global exception handler that applies to all controllers in the application.
  • Define methods within the class using @ExceptionHandler to catch specific exception types.
  • Return customized error messages, HTTP status codes, and optionally include stack traces or error codes.
  • Helps separate exception handling logic from controller code and ensures consistency in API responses.
@ControllerAdvice public class GlobalExceptionHandler { @ExceptionHandler(EntityNotFoundException.class) public ResponseEntity handleNotFound(EntityNotFoundException ex) { return ResponseEntity.status(HttpStatus.NOT_FOUND).body("Entity not found: " + ex.getMessage()); } }
34. What is the role of @Valid, @Validated, and custom validators?
  • @Valid is a JSR-303 annotation used to trigger validation on method parameters or bean fields.
  • @Validated is a Spring-specific annotation that provides group-based validation capabilities.
  • Both can be applied on controller method parameters, service layer arguments, or entity classes.
  • Custom validators can be created by implementing ConstraintValidator and annotated with a custom @Constraint.
  • Validation errors can be captured using BindingResult or MethodArgumentNotValidException.
public class UserDTO { @NotBlank(message = "Username is required") private String username; @Email(message = "Invalid email format") private String email; } @PostMapping("/register") public ResponseEntity registerUser(@Valid @RequestBody UserDTO user) { return ResponseEntity.ok("Registered"); }
35. How do you return custom error responses in REST?
  • Create a custom error response class to structure your error message, timestamp, and error details.
  • Return instances of this class from within your @ExceptionHandler methods.
  • Use ResponseEntity to include HTTP status codes alongside the custom body.
  • This improves client experience by standardizing error formats across endpoints.
public class ErrorResponse { private String message; private LocalDateTime timestamp; private String details; // Constructors, getters, setters } @ExceptionHandler(Exception.class) public ResponseEntity handleGeneric(Exception ex, WebRequest request) { ErrorResponse err = new ErrorResponse("Server error", LocalDateTime.now(), request.getDescription(false)); return new ResponseEntity<>(err, HttpStatus.INTERNAL_SERVER_ERROR); }
Security (Spring Security)
36. How do you secure a Spring Boot application?
  • Spring Boot integrates Spring Security by default when the dependency is added, securing all endpoints with HTTP Basic authentication.
  • You can override this behavior by configuring a SecurityFilterChain or extending WebSecurityConfigurerAdapter (pre-Spring Security 5.7).
  • Secure endpoints using annotations like @PreAuthorize or by defining custom access rules via the HttpSecurity DSL.
  • Supports authentication providers like in-memory, JDBC, LDAP, and OAuth2.
37. What is the difference between authentication and authorization?
  • Authentication verifies the identity of a user (e.g., username/password validation).
  • Authorization determines whether an authenticated user has permission to access a specific resource.
  • Authentication is typically handled by a login mechanism, while authorization is managed using roles and access control rules.
38. How to implement JWT-based authentication?
  • Use libraries like jjwt or auth0 for JWT handling.
  • Generate JWT upon successful login and send it in the Authorization header (Bearer token) for each request.
  • Create a filter that validates incoming JWTs and sets the SecurityContext.
  • JWTs should be signed and optionally encrypted for integrity and confidentiality.
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6...
39. What is method-level security (@PreAuthorize, @Secured)?
  • Method-level security restricts access to individual methods using annotations like @PreAuthorize and @Secured.
  • @PreAuthorize uses SpEL to express complex rules like roles or conditions on method parameters.
  • @Secured is simpler and only supports role-based access control.
  • Enable method security using @EnableMethodSecurity or @EnableGlobalMethodSecurity in the configuration class.
@PreAuthorize("hasRole('ADMIN')") public void deleteUser(Long id) {...}
40. How do you configure password encoders?
  • Password encoders are used to hash and verify passwords securely.
  • Use BCryptPasswordEncoder as it is recommended for its strength and adaptive hashing.
  • Define a PasswordEncoder bean in your configuration to integrate with the authentication provider.
@Bean public PasswordEncoder passwordEncoder() { return new BCryptPasswordEncoder(); }
Spring Boot Testing
41. What is @SpringBootTest vs @WebMvcTest vs @DataJpaTest?
  • @SpringBootTest: Loads the complete application context. Best for integration tests involving multiple layers (controller, service, repository).
  • @WebMvcTest: Loads only the web layer (controllers and related components), useful for focused unit testing of controllers.
  • @DataJpaTest: Configures an in-memory database and scans JPA repositories, ideal for testing JPA repository behavior in isolation.
42. How do you write integration tests for REST endpoints?
  • Use @SpringBootTest along with TestRestTemplate or MockMvc to perform HTTP requests and validate responses.
  • Annotate test classes with @AutoConfigureMockMvc when using MockMvc.
  • Write assertions against status codes, headers, and JSON responses.
  • Use @Transactional for tests involving database modifications to ensure data cleanup.
43. How do you mock services using @MockBean?
  • @MockBean is used to mock and inject a bean into the Spring ApplicationContext for testing.
  • Replaces the actual bean with a mock version in tests annotated with @SpringBootTest or @WebMvcTest.
  • Works seamlessly with Mockito to define expectations and behavior.
  • Ensures isolation of the component under test by removing dependencies on other layers.
@MockBean private UserService userService; when(userService.getUser()).thenReturn(new User("John"));
44. What is TestRestTemplate and when to use it?
  • TestRestTemplate is a convenient alternative to RestTemplate used specifically for integration testing in Spring Boot.
  • Supports synchronous HTTP calls and simplifies testing of REST APIs.
  • It is injected automatically in tests annotated with @SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT).
  • Can be used to validate full end-to-end flows from the controller down to the database layer.
@Autowired private TestRestTemplate restTemplate; ResponseEntity response = restTemplate.getForEntity("/api/hello", String.class); assertEquals(HttpStatus.OK, response.getStatusCode());
Spring Boot Actuator & Monitoring
45. What is Spring Boot Actuator? What endpoints are available?
  • Spring Boot Actuator provides production-ready features for monitoring and managing applications.
  • It exposes a set of built-in endpoints such as /actuator/health, /actuator/info, /actuator/metrics, /actuator/env, etc.
  • Custom endpoints can also be created using the @Endpoint or @ReadOperation annotations.
  • Endpoints can be enabled or disabled individually in application.properties.
46. How do you secure actuator endpoints?
  • Use Spring Security to restrict access to actuator endpoints.
  • Define access control rules in your SecurityFilterChain configuration.
  • For example, allow unauthenticated access to /actuator/health but secure others using roles.
  • Set the property management.endpoints.web.exposure.include to control visibility.
management.endpoints.web.exposure.include=health,info management.endpoints.web.exposure.exclude=env,beans
47. How do you customize health checks?
  • Implement the HealthIndicator interface and override health() method to provide custom logic.
  • Register your custom implementation as a Spring bean.
  • It will automatically be included in the output of /actuator/health.
  • Useful for checking dependencies like database connections, APIs, or disk space.
@Component public class MyServiceHealthIndicator implements HealthIndicator { @Override public Health health() { return Health.up().withDetail("myService", "Available").build(); } }
48. How to integrate Prometheus or Micrometer with Spring Boot?
  • Micrometer is the metrics facade integrated with Spring Boot Actuator.
  • Add the micrometer-registry-prometheus dependency to enable Prometheus support.
  • Prometheus scrapes metrics from /actuator/prometheus endpoint.
  • Configure Prometheus to collect data periodically and visualize it using Grafana.
io.micrometer micrometer-registry-prometheus
Logging & Metrics
49. How do you configure logging levels in Spring Boot?
  • Spring Boot uses application.properties or application.yml to configure logging levels.
  • You can specify log levels for packages or classes using logging.level prefix.
  • Supports levels: TRACE, DEBUG, INFO, WARN, ERROR, OFF.
logging.level.org.springframework=INFO logging.level.com.mycompany.myapp=DEBUG
50. What is the default logging framework used?
  • Spring Boot uses Logback as the default logging framework.
  • It provides built-in support for log file rotation, filtering, and formatting.
  • Other logging frameworks like Log4j2 or Java Util Logging can be used by excluding Logback and adding the respective dependency.
51. How do you integrate external loggers like Logback or Log4j2?
  • To use Logback: create a custom logback-spring.xml file in the classpath with your configuration.
  • To use Log4j2: exclude Spring Boot's default logging and add spring-boot-starter-log4j2 to your dependencies.
  • Spring Boot will automatically detect and use the provided configuration files (e.g., log4j2-spring.xml).
  • Allows customization of log patterns, appenders, and log rotation policies.
org.springframework.boot spring-boot-starter-log4j2
Spring Boot with Messaging Systems
52. How to integrate Kafka with Spring Boot?
  • Add the spring-kafka dependency to your project.
  • Configure Kafka properties in application.properties (e.g., bootstrap servers, topic names).
  • Use @KafkaListener to consume messages and KafkaTemplate to produce them.
  • Spring Boot auto-configures the Kafka producer and consumer factories using the provided configuration.
@KafkaListener(topics = "my-topic", groupId = "group_id") public void consume(String message) { System.out.println("Received: " + message); } @Autowired private KafkaTemplate kafkaTemplate; kafkaTemplate.send("my-topic", "Hello Kafka");
53. What is Spring Cloud Stream?
  • Spring Cloud Stream is a framework for building event-driven microservices using messaging middleware like Kafka or RabbitMQ.
  • Abstracts message brokers behind binders and provides a simple programming model for message publishing and consumption.
  • Uses @EnableBinding, @StreamListener, and functional binding model with suppliers and consumers.
  • Greatly simplifies messaging logic and promotes decoupled architecture.
54. How do you consume and produce messages using Spring Boot?
  • Use KafkaTemplate to publish messages to a Kafka topic.
  • Use @KafkaListener to annotate a method that handles incoming messages.
  • For RabbitMQ, use RabbitTemplate and @RabbitListener similarly.
  • Ensure correct serialization and deserialization strategies are defined via producer and consumer configs.
// Kafka Producer kafkaTemplate.send("topic-name", "event-data"); // Kafka Consumer @KafkaListener(topics = "topic-name") public void listen(String message) { log.info("Consumed: " + message); }
Spring Boot with Kafka & MQ (Advanced)
55. How to configure Kafka consumers and producers in Spring Boot?
  • Use application.properties or application.yml to configure producer and consumer properties.
  • Configure serializers/deserializers, group id, bootstrap servers, and concurrency.
  • Spring Boot auto-configures the KafkaTemplate and ConcurrentKafkaListenerContainerFactory.
# application.properties spring.kafka.bootstrap-servers=localhost:9092 spring.kafka.consumer.group-id=my-group spring.kafka.consumer.auto-offset-reset=earliest spring.kafka.producer.key-serializer=org.apache.kafka.common.serialization.StringSerializer spring.kafka.producer.value-serializer=org.apache.kafka.common.serialization.StringSerializer
56. What is a Kafka listener container?
  • It’s a Spring-managed component that listens to Kafka topics and delegates messages to @KafkaListener methods.
  • Supports concurrent message consumption using multiple threads (via concurrency setting).
  • Manages error handling, retries, and acknowledgment strategies internally.
  • Configured through ConcurrentKafkaListenerContainerFactory.
57. How to manage offset manually in Kafka?
  • Use ackMode = MANUAL in the container factory configuration.
  • Inject Acknowledgment in your @KafkaListener method and call ack.acknowledge() after successful processing.
  • This gives fine-grained control over when offsets are committed, useful in error-prone environments.
@KafkaListener(topics = "topic", containerFactory = "manualAckContainerFactory") public void listen(String msg, Acknowledgment ack) { process(msg); ack.acknowledge(); }
58. How do you achieve exactly-once delivery in Kafka?
  • Enable enable.idempotence=true on the producer to avoid duplicate sends.
  • Use transactional producers and set transactional.id to ensure atomicity.
  • Wrap both producer send and consumer offset commit in a single Kafka transaction.
  • Ensure idempotent processing logic to handle duplicates gracefully.
producer.initTransactions(); producer.beginTransaction(); producer.send(...); producer.commitTransaction();
59. How to handle message retries and dead letter queues?
  • Configure SeekToCurrentErrorHandler with retry backoff and max attempts.
  • Use DeadLetterPublishingRecoverer to redirect failed messages to a dead-letter topic after retries are exhausted.
  • Dead letter topics help in isolating bad messages for later inspection or reprocessing.
  • Enable EnableKafka and configure a retry-aware KafkaListenerContainerFactory.
factory.setErrorHandler(new SeekToCurrentErrorHandler( new DeadLetterPublishingRecoverer(kafkaTemplate), new FixedBackOff(1000L, 3)));
Concurrency & Multi-threading (Advanced)
60. How do you create async processes in Spring Boot?
  • Use the @Async annotation to run methods asynchronously in the background thread.
  • Enable asynchronous processing using @EnableAsync in a configuration class.
  • Return Future, CompletableFuture, or void depending on the requirement.
61. What is @Async? How do you configure Executor?
  • @Async is a Spring annotation that allows the execution of a method in a separate thread.
  • To configure the thread pool executor, define a TaskExecutor bean with desired pool size and queue capacity.
  • Set the name of the executor with @Async("executorName") to use a specific bean.
@Bean(name = "asyncExecutor") public Executor taskExecutor() { ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor(); executor.setCorePoolSize(5); executor.setMaxPoolSize(10); executor.setQueueCapacity(500); executor.setThreadNamePrefix("Async-"); executor.initialize(); return executor; }
62. How do you schedule jobs with @Scheduled?
  • Use the @Scheduled annotation to schedule tasks at fixed rate, delay, or using cron expressions.
  • Enable scheduling with @EnableScheduling annotation on the configuration class.
  • Useful for running periodic tasks like report generation, cleanup jobs, etc.
@Scheduled(fixedRate = 5000) public void runTask() { System.out.println("Running scheduled task"); }
63. How do you manage concurrent access to shared resources?
  • Use Java concurrency utilities like synchronized, ReentrantLock, or ReadWriteLock.
  • Use database locks for distributed systems — optimistic (versioning) or pessimistic locks.
  • For in-memory caches, use concurrent data structures like ConcurrentHashMap.
64. What are ThreadPoolTaskExecutor and ScheduledThreadPoolExecutor?
  • ThreadPoolTaskExecutor is a Spring-managed thread pool for asynchronous method execution.
  • Allows configuration of core size, max size, queue capacity, and rejection policy.
  • ScheduledThreadPoolExecutor is a JDK executor for scheduled tasks supporting delay and periodic execution.
  • Can be used for custom scheduling instead of @Scheduled.
65. How do you implement rate-limiting or throttling?
  • Use token bucket or leaky bucket algorithm to limit requests per time unit.
  • Apply filters or interceptors at the controller level.
  • Use libraries like Bucket4j or Resilience4j RateLimiter.
  • For distributed apps, use Redis or other external stores to maintain request counters.
66. How do you handle concurrent DB writes safely?
  • Use optimistic locking via @Version in JPA entities to detect and prevent concurrent update conflicts.
  • Use pessimistic locking (e.g., PESSIMISTIC_WRITE) for critical sections.
  • Ensure atomic operations using database transactions and isolation levels.
@Version private Long version; // Enables optimistic locking in JPA
Microservices Patterns
67. How do you implement service discovery (Eureka, Consul)?
  • Use Spring Cloud Netflix Eureka or Spring Cloud Consul for dynamic service discovery.
  • Register microservices with the discovery server using @EnableEurekaClient or @EnableDiscoveryClient.
  • Use DiscoveryClient or service names in Feign clients to resolve service addresses dynamically.
68. How do you use Spring Cloud Config?
  • Spring Cloud Config provides centralized external configuration management for microservices.
  • Set up a config server with Git or filesystem backend.
  • Use @EnableConfigServer on the server and spring.cloud.config.uri on the clients.
  • Supports dynamic refresh of configurations using Spring Cloud Bus and @RefreshScope.
69. What is circuit breaker pattern? How do you use Resilience4j?
  • A circuit breaker prevents cascading failures in distributed systems by stopping attempts to reach failing services.
  • Resilience4j is a lightweight fault tolerance library offering circuit breakers, rate limiters, retries, etc.
  • Use @CircuitBreaker annotation to wrap service calls and fallback logic for failure handling.
@CircuitBreaker(name = "backendService", fallbackMethod = "fallback") public String callBackend() { return restTemplate.getForObject("http://service/api", String.class); }
70. What is an API Gateway? How do you use Spring Cloud Gateway?
  • An API Gateway routes requests to appropriate services and handles cross-cutting concerns like logging, rate-limiting, and security.
  • Spring Cloud Gateway is a non-blocking, reactive gateway built on Spring WebFlux.
  • Define routes and predicates in application.yml or Java config to route traffic dynamically.
71. How do you trace requests across services (Zipkin/Sleuth)?
  • Spring Cloud Sleuth adds trace and span IDs to logs for distributed tracing.
  • Use Zipkin to collect and visualize trace data from multiple services.
  • Enable tracing by including Sleuth and Zipkin dependencies; traces are auto-propagated over HTTP and messaging systems.
72. What is service orchestration vs choreography?
  • Orchestration: A central service coordinates tasks across services (e.g., using Camunda, BPM).
  • Choreography: Services react to events without a central controller, ensuring loose coupling (event-driven architecture).
  • Orchestration suits complex workflows, while choreography fits reactive, scalable systems.
Deployment & CI/CD
73. How to create a Docker image for Spring Boot app?
  • Create a Dockerfile in the root of your Spring Boot project.
  • Use a base image like openjdk or adoptopenjdk and copy your JAR file into the image.
  • Expose the port your application runs on and define the entry point with ENTRYPOINT.
  • Build the image using docker build -t myapp . and run it with docker run.
FROM openjdk:17-jdk-alpine COPY target/app.jar app.jar ENTRYPOINT ["java", "-jar", "/app.jar"]
74. How to run Spring Boot in Kubernetes?
  • Package the Spring Boot app as a Docker image and push it to a container registry (e.g., Docker Hub, ECR).
  • Create Kubernetes deployment and service YAML files specifying the image, replicas, ports, etc.
  • Use kubectl apply -f deployment.yaml to deploy your app.
  • Optionally use Helm charts or Kustomize for templated deployments and versioning.
75. How to use profiles for environment-specific configurations in deployment?
  • Use application-{profile}.properties or .yml files to define environment-specific settings.
  • Activate profiles using SPRING_PROFILES_ACTIVE environment variable or --spring.profiles.active command line argument.
  • Each profile can override base configuration, allowing flexibility in dev, QA, staging, and prod environments.
76. What are the best practices for production configuration?
  • Enable health checks, monitoring, and metrics via Spring Boot Actuator.
  • Use environment variables or externalized configuration to avoid hardcoding secrets.
  • Enable secure HTTPS, CSRF protection, and role-based access control.
  • Use proper logging configuration with file rotation and log aggregation.
  • Test configuration changes in staging before promoting to production.
Performance Tuning
77. How to tune connection pools (HikariCP)?
  • HikariCP is the default connection pool used in Spring Boot (since 2.0).
  • Configure properties like maximumPoolSize, minimumIdle, idleTimeout, and maxLifetime in application.properties.
  • Monitor connection pool metrics using Spring Boot Actuator or JMX.
  • Avoid setting max pool size too high; start with number of cores * 2.
spring.datasource.hikari.maximum-pool-size=10 spring.datasource.hikari.idle-timeout=30000 spring.datasource.hikari.max-lifetime=1800000
78. How to detect memory leaks or CPU spikes in Spring Boot?
  • Use monitoring tools like VisualVM, JConsole, or Java Flight Recorder (JFR) to inspect JVM performance.
  • Enable GC logs and monitor heap usage, GC frequency, and thread dumps.
  • Integrate Spring Boot Actuator with Prometheus + Grafana for observability.
  • Check for unclosed resources and thread leaks in long-running services.
79. How to optimize startup time and reduce memory footprint?
  • Remove unnecessary dependencies and auto-configurations using @SpringBootApplication(exclude = ...).
  • Use lazy initialization by setting spring.main.lazy-initialization=true.
  • Reduce log level and avoid heavy log operations during startup.
  • Precompile configuration using tools like Spring AOT or GraalVM native image (for extreme cases).
